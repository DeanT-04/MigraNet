## 🎨 Gephi配置详解

### 布局算法: ForceAtlas 2

#### 核心参数:
```
斥力强度: 2000    (默认值的2倍，避免节点重叠)
重力: 50         (默认值的10倍，保持紧凑)
防止重叠: ✅ 开启 (强制要求)
边权重影响: 1.0   (完全考虑边权重)
运行时间: 3-5分钟 (直到网络稳定)
```

#### 为什么这样设置？
- **高斥力**: 238个节点需要足够空间
- **高重力**: 防止网络过于分散
- **防止重叠**: 确保标签可读
- **边权重**: 强连接的节点更靠近

---

### 外观设置

#### 节点样式
```
颜色: 按 Category 字段分类
  - trigger_mechanisms  → 黄色
  - true_comorbidities  → 红色
  - social_impact       → 蓝色
  - interventions       → 绿色

大小: 按 Frequency 字段
  - 最小值: 2
  - 最大值: 15
  - 算法: 线性缩放

标签: 只显示 Frequency > 5 的节点
  - 减少视觉混乱
  - 突出重要节点
```

#### 边样式
```
透明度: 0.3 (30%)
  - 提高可读性
  - 减少重叠遮挡

粗细: 按 Weight 字段
  - 强关联更粗
  - 弱关联更细

颜色: 灰色或继承源节点颜色
```

## 🎯 使用步骤

### 1. 导入数据
```
Gephi → 打开 → 导入节点和边CSV文件
```

### 2. 应用布局
```
布局面板 → 选择 ForceAtlas 2 → 设置参数 → 运行
```

### 3. 设置外观
```
外观面板 → 节点 → 颜色 → 按分区 → Category
外观面板 → 节点 → 大小 → 按排名 → Frequency
```

### 4. 优化标签
```
标签面板 → 字体大小: 12pt
标签调整 → 防止重叠
显示标签 → 仅高频节点
```

### 5. 导出图像
```
预览 → 调整视角 → 导出为PNG/SVG
推荐尺寸: 4000x3000px (300 DPI)
```

## 🔧 故障排除

### 问题: 节点仍然重叠
**解决**: 增加斥力强度到3000-4000

### 问题: 网络过于分散
**解决**: 增加重力到100-150

### 问题: 标签难以阅读
**解决**: 
- 增大字体
- 提高Frequency阈值（只显示>10的节点）

### 问题: 布局运行太慢
**解决**: 
- 减少到3分钟
- 使用简化版布局（YifanHu）
